NeoPress Docs

NeoPress — Anime & Gaming Media Platform

A futuristic, dark-mode editorial hub for spotlighting anime, games, and tech culture. This guide breaks down the experience, architecture, and key features powering the NeoPress CMS.

Product Overview

NeoPress blends a bento-grid editorial homepage with a cinematic article experience. A custom admin CMS lets editors publish rich content that syncs locally and can optionally sync to Supabase.

  • Hero carousel with featured stories, tags, and spotlight author details.
  • Editorial grid that highlights posts with wide and tall card variations.
  • Rich article template featuring read time, author, and related content.

Core Features

  • Local-first publishing: Posts are written and stored via localStorage.
  • Rich editor: H1/H2 headings, bold/italic formatting, links, and image embeds.
  • Admin tools: Secure login, content dashboard, create/edit/delete flow.
  • Optional Supabase sync: Publish to a posts table when credentials exist.

Structure & Key Files

  • src/pages/Home.jsx — Hero carousel + grid feed.
  • src/pages/Article.jsx — Article template, read time, author card.
  • src/pages/AdminEditor.jsx — Rich editor + publishing flow.
  • src/data/mockPosts.js — Seed content for the UI.
  • src/utils/storage.js — Local storage helpers.
  • src/lib/supabaseClient.js — Optional database integration.

How To Run

  • Install dependencies: npm install
  • Run dev server: npm run dev
  • Build production: npm run build
  • Preview build: npm run preview

Deploying on Replit

  • Create a new Replit project and import this repository.
  • Set the run command to npm install && npm run dev -- --host 0.0.0.0 --port 3000.
  • Expose port 3000 in Replit so the Vite dev server is reachable.
  • Optional: add VITE_SUPABASE_URL and VITE_SUPABASE_ANON_KEY in Secrets.
